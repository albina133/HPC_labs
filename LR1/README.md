# **Лабораторная работа. Перемножение матриц**
В работе реализован алгоритм умножения матриц двумя способами: на CPU и на GPU с использованием CUDA. Программа вычисляет произведение двух матриц, измеряет время выполнения, проверяет корректность результатов и оценивает ускорение.

## Описание реализации

### CUDA-ядро: `matrixMulKernel`

**Реализация:**
- каждый поток отвечает за один элемент результирующей матрицы;
- координаты элемента определяются через индексы потока;
- выполняется цикл по общей размерности матриц;
- результат записывается в соответствующую ячейку.

**Идея параллелизации** заключается в том, что все элементы матрицы считаются независимо, поэтому можно одновременно запускать большое количество потоков, что позволяет значительно ускорить вычисления на GPU.

### CPU: `matrix_multiply_cpu`
**Реализация:**
- алгоритм с тремя вложенными циклами;
- внешние циклы проходят по строкам и столбцам результирующей матрицы;
- каждый элемент вычисляется как скалярное произведение строки матрицы A и столбца матрицы B.

### GPU: `matrix_multiply_gpu`
**Реализация:**
- на GPU выделяется память под входные и выходную матрицы;
- данные копируются с CPU на GPU;
- запускается cuda-ядро, которое выполняет вычисления;
- после завершения результат копируется обратно;
- разбиение на блоки по 16x16 потоков.

### Фунция проверки: `check`
**Реализация:**
- проверка выполняется по каждому элементу;
- допускается небольшая погрешность.

### Основная функция: `main`
**Реализация:**
- создаются и заполняются матрицы;
- вызываются реализации GPU и CPU;
- выводится время выполнения, корректность и ускорение.

## Что конкретно было распараллелено и **почему**?
В параллельной версии алгоритма было распараллелено вычисление элементов результирующей матрицы. <br>
Каждый элемент матрицы C вычисляется независимо по формуле: <br>
<img width="1007" height="185" alt="image" src="https://github.com/user-attachments/assets/46d957a2-6307-468b-8c0c-c6e97e1a0d8a" />
- Каждому потоку GPU соответствует один элемент матрицы C.
- Поток определяет свои индексы (i, j) и вычисляет значение C[i][j].
- Все потоки выполняются одновременно.

То есть распараллелена основная вычислительная часть алгоритма - это подсчёт элементов результирующей матрицы. Это возможно потому, что вычисление каждого элемента не зависит от других, нет необходимости в обмене данными между потоками, а также отсутствуют конфликты записи в память.<br> 
Поэтому этот подход позволяет задействовать большое количество потоков и существенно сократить время выполнения по сравнению с последовательной реализацией на CPU.

## Выводы
В ходе экспериментов было проведено сравнение времени выполнения умножения матриц на CPU и GPU для различных размеров. <br>
Полученные результаты показывают, что:
- При малых размерах матриц (128x128, 256x256) CPU работает быстрее GPU из-за накладных расходов на выделение памяти, копирование данных и запуск CUDA-ядра. <br>
- Начиная с размера 512x512 GPU начинает показывать выигрыш по времени.
- При увеличении размера матриц ускорение существенно растёт и для матрицы 2000x2000 достигает более чем 270 раз.

Использование GPU эффективно для задач с большим объёмом вычислений, где затраты на подготовку данных становятся незначительными по сравнению с временем самих вычислений.
