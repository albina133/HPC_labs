# **Лабораторная работа. Сумма элементов вектора**
В работе реализован алгоритм сложения элементов вектора двумя способами: на CPU и на GPU с использованием CUDA. Программа вычисляет сумму элементов вектора, измеряет время выполнения для каждой реализации и оценивает ускорение GPU относительно CPU

## Описание реализации

### CUDA-ядро: `sumKernel`

**Реализация:**
- каждый поток GPU обрабатывает один элемент входного вектора;
- поток считывает свое значение из глобальной памяти;
- далее значения потоков внутри блока сохраняются в `shared memory`;
- после этого внутри блока выполняется редукция, то есть поэтапное суммирование элементов;
- первый поток блока добавляет частичную сумму блока в общий результат с помощью `atomicAdd`.

**Идея параллелизации** заключается в том, что элементы вектора на первом этапе можно обрабатывать независимо друг от друга. Каждый поток считывает свой элемент, а затем выполняется параллельное суммирование внутри блока. Такой подход позволяет задействовать большое количество потоков GPU одновременно

### CPU: `get_sum_cpu`
**Реализация:**
- используется последовательный алгоритм;
- выполняется один цикл по всем элементам вектора;
- каждый элемент последовательно прибавляется к общей сумме.

### GPU: `get_sum_gpu`
**Реализация:**
- на GPU выделяется память под входной вектор и итоговую сумму;
- входные данные копируются с CPU на GPU;
- запускается CUDA-ядро `sumKernel`;
- после завершения вычислений результат копируется обратно на CPU;
- время работы измеряется с учетом выделения памяти, копирования данных, выполнения ядра и получения результата.

### Основная функция: `main`
**Реализация:**
- создается вектор заданного размера;
- вектор заполняется случайными числами в диапазоне от 1 до 9;
- вызывается функция вычисления суммы на CPU;
- вызывается функция вычисления суммы на GPU;
- выводятся сумма, время выполнения и ускорение.

## Что конкретно было распараллелено и **почему**?
В параллельной версии алгоритма было распараллелено сложение элементов вектора.  
На первом этапе каждый поток GPU независимо считывает один элемент массива и записывает его во временный буфер блока в `shared memory`.

Далее внутри каждого блока выполняется редукция:
- на каждом шаге количество активных потоков уменьшается в 2 раза;
- потоки суммируют пары значений;
- в итоге в каждом блоке получается одна частичная сумма.

После этого:
- первый поток каждого блока добавляет частичную сумму блока в общий результат;
- для корректного обновления общей переменной используется `atomicAdd`.

То есть распараллелена основная вычислительная часть алгоритма - суммирование большого количества элементов. Это возможно, потому что на начальном этапе чтение и обработка элементов не зависят друг от друга. Независимость операций позволяет эффективно распределить работу между потоками GPU

- Язык реализации: C++ / CUDA

## Выводы
В ходе экспериментов было проведено сравнение времени выполнения суммы элементов вектора на CPU и GPU для различных размеров. <br>
Полученные результаты показывают, что:
- При малых размерах вектора (`1000`, `10000`, `50000`) CPU работает значительно быстрее GPU из-за накладных расходов на выделение памяти, копирование данных и запуск CUDA-ядра. <br>
- Даже при увеличении размера вектора до `1000000` элементов GPU в данной реализации не показал выигрыша по времени, однако значение ускорения постепенно возрастало. <br>
- Время выполнения на GPU во всех экспериментах оставалось примерно на одном уровне (`0.28–0.31` сек), тогда как время выполнения на CPU увеличивалось с ростом размера входных данных. <br>

Таким образом, для задачи суммирования элементов вектора в данной реализации использование GPU не оказалось эффективнее CPU, так как накладные расходы на подготовку и передачу данных превышают выигрыш от параллельного выполнения вычислений.

## Дополнительный эксперимент
Также был проведен эксперимент на векторе размером `50000000` элементов. <br>
В результате были получены следующие значения:
- CPU: `0.739799` сек
- GPU: `0.359174` сек
- Ускорение: `2.05972` <br>

Этот эксперимент показывает, что при очень больших объемах данных GPU начинает работать эффективнее CPU.  
В данном случае накладные расходы на выделение памяти, копирование данных и запуск CUDA-ядра становятся менее значительными по сравнению со временем самих вычислений. <br>

Использование GPU особенно оправдано для задач с большим количеством входных данных, где параллельная обработка позволяет получить заметный выигрыш по времени
