# Bilateral filtering on GPU and CPU

## Цель работы

Реализация **9-точечного bilateral-фильтра** для полутонового изображения с использованием **CUDA**.

Необходимо было:
- взять входное изображение в формате **BMP**;
- использовать значения параметров `sigma_d` и `sigma_r`;
- реализовать **9-point bilateral filter**, то есть фильтр с окном **3x3**;
- выполнить обработку изображения на **GPU**;
- обязательно использовать **texture memory** для чтения пикселей;
- реализовать обработку граничных пикселей через ближайшие допустимые значения;
- реализовать также CPU-версию фильтра для сравнения;
- сохранить результирующие изображения в формате **BMP**;
- вывести время обработки на **GPU** и на **CPU**.

## Выполнение задания

В программе выполнены следующие шаги:

1. определяется доступное CUDA-устройство;
2. загружается исходное изображение `thai.bmp`;
3. изображение читается вручную через структуры BMP-заголовков;
4. входное изображение переводится в полутоновый формат;
5. grayscale-изображение сохраняется в файл `input_gray.bmp` для проверки входных данных;
6. входной массив пикселей копируется в CUDA array на GPU;
7. CUDA array привязывается к **texture memory**;
8. для каждого пикселя через `tex2D` считываются 9 значений из окна **3x3**;
9. для краевых пикселей координаты ограничиваются ближайшим допустимым пикселем;
10. для каждого пикселя вычисляются веса bilateral-фильтра;
11. результат сохраняется в выходной массив на GPU;
12. результат копируется обратно на CPU;
13. дополнительно выполняется такая же фильтрация на CPU;
14. сохраняются изображения `bilateral_gpu.bmp` и `bilateral_cpu.bmp`;
15. выводится время обработки на GPU и CPU.

## Формула фильтра

Для каждого пикселя вычисляется новое значение интенсивности:

```text
h(a0) = 1 / k * sum(f(ai) * g(ai) * r(ai))
```

где `k` - нормировочный коэффициент:

```text
k = sum(g(ai) * r(ai))
```

В программе используются два веса:

```text
g(x, y) = exp(-(x^2 + y^2) / sigma_d^2)
```

`g(x, y)` зависит от расстояния соседнего пикселя до центрального.

```text
r(ai) = exp(-((f(ai) - f(a0))^2) / sigma_r^2)
```

`r(ai)` зависит от разницы интенсивностей между соседним и центральным пикселем.

Таким образом, bilateral-фильтр сглаживает изображение, но лучше сохраняет резкие границы объектов, чем обычное гауссово размытие.

## Реализация в коде

Используются следующие основные части программы.

### BMP-заголовки

Для работы с BMP используются структуры:

```cpp
struct BmpFileHeader
struct BmpInfoHeader
```

Они описывают файловый и информационный заголовки BMP-файла. Для корректного чтения используется:

```cpp
#pragma pack(push, 1)
```

Это отключает автоматическое выравнивание полей структуры компилятором.

### Хранение изображения

Для хранения входного изображения используется структура:

```cpp
struct GrayImage
```

Она содержит:
- `width` - ширину изображения;
- `height` - высоту изображения;
- `data` - одномерный массив пикселей grayscale-изображения.

### Проверка CUDA-вызовов

Для проверки ошибок CUDA используется функция:

```cpp
checkCuda()
```

Она принимает результат CUDA-вызова и текст сообщения. Если CUDA-функция завершилась с ошибкой, программа выбрасывает исключение и выводит сообщение об ошибке.

Это используется, например, при:
- выборе CUDA-устройства;
- выделении памяти;
- создании texture object;
- запуске kernel-функции;
- копировании результата обратно на CPU;
- освобождении ресурсов.

### Ограничение координат

Для обработки границ изображения используются функции:

```cpp
clampHost()
clampDeviceInt()
```

`clampHost()` используется на CPU, а `clampDeviceInt()` используется внутри CUDA kernel.

Если окно **3x3** выходит за границы изображения, координаты ограничиваются ближайшим допустимым пикселем. Например:
- если `x < 0`, берётся `x = 0`;
- если `x >= width`, берётся `x = width - 1`;
- если `y < 0`, берётся `y = 0`;
- если `y >= height`, берётся `y = height - 1`.

Это соответствует требованию брать недостающие значения для краевых строк и столбцов из ближайших пикселей.

### Информация о GPU

Функция:

```cpp
printCudaDeviceInfo()
```

выводит информацию о доступном CUDA-устройстве:
- количество CUDA-устройств;
- название выбранной видеокарты;
- compute capability;
- объём global memory.

В программе выбирается первое доступное CUDA-устройство.

### Чтение BMP-файла

Функция:

```cpp
readBmpGray()
```

считывает BMP-файл и возвращает полутоновое изображение.

Поддерживаются изображения:
- **8-bit BMP**;
- **24-bit BMP**;
- **32-bit BMP**.

Если изображение уже 8-битное, оно считывается через палитру. Если изображение цветное, то есть 24-bit или 32-bit, оно переводится в grayscale по формуле яркости:

```text
gray = 0.299 * R + 0.587 * G + 0.114 * B
```

Также учитывается порядок строк в BMP. Если высота положительная, строки в BMP-файле хранятся снизу вверх, поэтому при чтении они переводятся в обычный порядок сверху вниз.

### Сохранение BMP-файлов

Функция:

```cpp
writeBmpGray()
```

сохраняет полутоновое изображение в формате **8-bit BMP**.

При сохранении создаётся grayscale-палитра, где:

```text
0   = черный
255 = белый
```

Также учитывается выравнивание строк BMP до кратности 4 байтам.

Программа сохраняет три изображения:

- `input_gray.bmp` - входное изображение после чтения и перевода в grayscale;
- `bilateral_gpu.bmp` - результат bilateral-фильтра на GPU;
- `bilateral_cpu.bmp` - результат bilateral-фильтра на CPU.

### CPU-версия фильтра

Функция:

```cpp
applyBilateralFilterCpu()
```

выполняет bilateral-фильтр на CPU.

Для каждого пикселя:
- берётся окно **3x3**;
- формируется массив из 9 соседних значений;
- отдельно сохраняются смещения `dx` и `dy`;
- для краёв используется ближайший допустимый пиксель;
- вычисляется центральный пиксель `f(a0)`;
- вычисляются веса `g` и `r`;
- считается нормировочный коэффициент `k`;
- итоговое значение записывается в выходной массив.

Время CPU-обработки измеряется через:

```cpp
std::chrono::high_resolution_clock
```

CPU-версия нужна для сравнения времени выполнения с GPU.

### CUDA kernel

Основная обработка на GPU выполняется в kernel-функции:

```cpp
__global__ void bilateralFilterTexture(...)
```

В этой функции каждый поток обрабатывает один пиксель изображения.

Внутри kernel:
- определяется координата текущего пикселя `x`, `y`;
- выполняется проверка выхода за пределы изображения;
- центральный пиксель `f(a0)` считывается через texture memory;
- 8 соседних пикселей также считываются через `tex2D`;
- всего используется 9 значений из окна **3x3**;
- при выходе за границы изображения координаты ограничиваются ближайшим допустимым пикселем;
- рассчитываются веса bilateral-фильтра;
- считается нормировочный коэффициент `k`;
- итоговое значение округляется и ограничивается диапазоном от `0` до `255`;
- результат записывается в выходной массив.

Чтение пикселей выполняется через:

```cpp
tex2D<unsigned char>(...)
```

Это означает, что CUDA-реализация использует **texture memory**, как и требуется в задании.

### Texture memory

Функция:

```cpp
applyBilateralFilterGpuTexture()
```

подготавливает данные для обработки на GPU.

В ней выполняются основные этапы CUDA-части:

1. создаётся описание формата элемента текстуры:

```cpp
cudaChannelFormatDesc channelDesc = cudaCreateChannelDesc<unsigned char>();
```

2. выделяется CUDA array:

```cpp
cudaMallocArray(&cuArray, &channelDesc, width, height)
```

3. входное grayscale-изображение копируется в CUDA array:

```cpp
cudaMemcpy2DToArray(...)
```

4. создаётся описание ресурса:

```cpp
cudaResourceDesc resDesc;
```

5. задаются параметры текстуры:

```cpp
cudaTextureDesc texDesc;
```

6. создаётся texture object:

```cpp
cudaCreateTextureObject(&texObj, &resDesc, &texDesc, 0)
```

7. выделяется память под выходное изображение на GPU;

8. запускается CUDA kernel;

9. результат копируется обратно на CPU;

10. освобождаются CUDA-ресурсы.

Для текстуры используются параметры:

```cpp
texDesc.addressMode[0] = cudaAddressModeClamp;
texDesc.addressMode[1] = cudaAddressModeClamp;
texDesc.filterMode = cudaFilterModePoint;
texDesc.readMode = cudaReadModeElementType;
texDesc.normalizedCoords = 0;
```

`cudaAddressModeClamp` соответствует обработке границ через ближайшие пиксели.

`cudaFilterModePoint` означает, что чтение выполняется без интерполяции, то есть берётся конкретное значение пикселя.

`normalizedCoords = 0` означает, что координаты текстуры задаются в обычных координатах изображения, а не в нормализованном диапазоне от `0` до `1`.

### Измерение времени

Время работы GPU-части измеряется с помощью CUDA events:

```cpp
cudaEvent_t startEvent;
cudaEvent_t stopEvent;
```

Измеряется время выполнения CUDA kernel.

После выполнения kernel вычисляется время:

```cpp
cudaEventElapsedTime(&elapsedMs, startEvent, stopEvent)
```

Время CPU-части измеряется отдельно через:

```cpp
std::chrono::high_resolution_clock
```

## Работа программы

По умолчанию программа использует файл:

```text
thai.bmp
```

и параметры:

```text
sigma_d = 2.0
sigma_r = 60.0
```

Также можно передать имя файла и значения sigma через аргументы командной строки:

```text
BilateralCuda.exe input.bmp 2.0 60.0
```

где:
- `input.bmp` - входное изображение;
- `2.0` - значение `sigma_d`;
- `60.0` - значение `sigma_r`.

После выполнения создаются файлы:

- `input_gray.bmp`;
- `bilateral_gpu.bmp`;
- `bilateral_cpu.bmp`.

## Выводы

В ходе лабораторной работы была реализована программа bilateral-фильтрации изображения на GPU с использованием CUDA. Был реализован 9-точечный bilateral-фильтр с окном **3x3**, в котором каждый пиксель обрабатывается независимо.

Для чтения центрального и соседних пикселей использовалась **texture memory**, что соответствует требованиям задания. Также была реализована обработка граничных пикселей: если окно фильтра выходит за пределы изображения, используется ближайший допустимый пиксель.

Программа сохраняет входное grayscale-изображение, результат обработки на GPU и результат обработки на CPU в формате BMP.

По результатам тестирования изображение размером `1440 x 1080` было обработано на GPU за `0.004598 seconds`, а на CPU за `0.499236 seconds`. Таким образом, обработка на GPU оказалась примерно в `108.58` раза быстрее.

В результате была успешно изучена работа с CUDA-приложениями, texture memory и параллельной обработкой изображений.
